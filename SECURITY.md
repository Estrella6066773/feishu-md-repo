# 安全与保密说明

Feishu MD Repo 是**本地桌面应用**：核心服务默认只监听 `127.0.0.1`，飞书凭证与同步状态保存在本机 SQLite，**不会**由本项目托管到云端。

## 敏感数据存放位置

| 内容 | 位置 | 是否应进入 Git |
|------|------|----------------|
| 飞书 `app_id` / `app_secret` | 本地 `app.db`（管理面板写入） | **否** |
| 绑定配置、Wiki/Drive 节点映射 | 同上 | **否** |
| 同步日志、机器人播报目标 | 同上 | **否** |
| 被同步的 Git 仓库内容 | 用户指定的本地/远程仓库 | **否**（除非你自己把业务仓库与本项目混放） |
| 环境变量 `.env` | 可选，仅端口/数据目录 | **否** |

默认数据目录：

- Windows：`%AppData%/Roaming/feishu-md-repo/app.db`
- 可通过 `FEISHU_MD_DATA_DIR` 覆盖

## 切勿提交的内容

提交前请确认未包含：

- `.env`、`.env.local` 及任何含真实密钥的文件
- `*.db`、`*.sqlite` 等数据库文件
- 导出的飞书凭证、Cookie、Token 截图或 JSON
- 含 `app_secret`、`tenant_access_token` 的调试日志
- 个人 `.cursor/` 规则（已在 `.gitignore` 中排除）

仓库提供检查脚本（见下方「提交前自检」）。

## 凭证泄露后的处理

若 `app_secret` 或数据库文件曾误提交、误分享：

1. 立即在 [飞书开放平台](https://open.feishu.cn/app) **重置 App Secret**
2. 在本应用「设置」中重新保存凭证
3. 若已推送到远程 Git：除重置密钥外，还需从历史中清除敏感文件（如 `git filter-repo`），并强制推送前评估协作影响
4. 检查飞书应用审计日志，确认无异常 API 调用

## 运行安全建议

- **不要**将 `FEISHU_MD_HOST` 改为 `0.0.0.0` 并对公网暴露，除非你在前方加了认证与 TLS
- 机器人指令白名单（`chat_id` / `open_id`）建议在生产环境配置，避免任意群成员触发同步
- 定期备份 `%AppData%/Roaming/feishu-md-repo/`，备份文件同样视为敏感数据
- 共享屏幕或录屏时，避免打开含 `app_secret` 的设置页

## 漏洞反馈

本项目为个人/团队本地工具，暂无公开漏洞赏金计划。若发现安全问题，请通过仓库 Issue（**勿在 Issue 中粘贴真实密钥**）或私下联系维护者说明复现步骤与影响范围。
